# Relationship / Gray-Traffic Social Video Notes

Use this reference when a short video is about dating, partner screening, sexual anxiety, gender tactics, PUA-style advice, or emotionally charged relationship monetization.

## Routing

- Detailed note path: `<your_vault_root>/情感知识库/视频解析报告/`
- Index row remains in: `<your_vault_root>/自媒体选题库/视频内容总表.md`
- Wikilink form: `[[情感知识库/视频解析报告/<note_name>]]`
- If the parser generated a default note under `自媒体选题库/视频解析报告/`, rewrite/move the final structured version into `情感知识库/视频解析报告/` and remove the duplicate default note.

## Default value classification

Prefer one of:

- `反面案例 + 知识库积累`: manipulative, classist, sexist, objectifying, anxiety-driven, or low-evidence advice.
- `知识库积累 + 二创素材`: useful structure can be migrated to healthier domains, but the original relationship advice should not be copied directly.
- `仅归档/低价值`: mostly recycled clichés or pure provocation.

## Report sections to add beyond transcript

1. `一句话总结`: factual, not endorsing the advice.
2. `核心观点拆解`: separate claimed mechanism from evidence.
3. `内容结构 / 流量套路`: identify pain hook, specific script, visual standard, authority wrapper, controversy bait.
4. `可复用价值`: state whether it is a reverse-engineering case, a reusable content formula, or low value.
5. `可迁移到本人账号的角度`: translate the structure into your own domain examples (research, study, productivity, business, self-management, etc.) when useful.
6. `风险 / 备注`: explicitly flag value-risk and factual uncertainty.

## Common risk labels

- 阶层偏见 / 物质化择偶
- 性别刻板 / 物化对象
- PUA / 操控型话术 / 以亲密为包装套信息
- 边界风险 / 不尊重对方隐私
- 经验断言，缺少证据
- 焦虑营销 / 私域转化诱导

## Safer migration pattern

Do not repeat manipulative relationship advice as positive instruction. Instead, migrate the **structure**:

- "通过隐蔽细节判断家境" → "通过公开信息和行为细节判断导师/团队是否靠谱"
- "筛选伴侣资源" → "筛选合作伙伴/项目/客户的可靠性"
- "话术套信息" → "透明、尊重边界的信息收集清单"

Always rewrite into ethical, domain-appropriate language before suggesting二创。
